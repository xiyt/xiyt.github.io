[最新]AWS免费一年云服务申请细节分享及配置Shadowsocks服务

AWS对新用户提供免费一年的云主机服务，相比大家早就听说过了。大家可以利用这个免费服务不管是自己搭建学习平台，还是搭建诸如Shadowsocks的工具类服务，都是一个很不错的选择。由于我之前已经在使用Vultr了，所以AWS这个免费资格一直没有使用，知道最近Vultr即将到期，我才在最近成功注册了AWS的免费服务 ，并配置好了Shadowsocks。下面我就把我的注册细节分享给大家。这个流程还是比较复杂的，请大家注意看关键点:-D

1、注册

注册地址：https://portal.aws.amazon.com/gp/aws/developer/registration/index.html?nc2=h_ct

通过这个地址打开注册页面，这里填写邮箱地址或手机号，然后选择我是新用户，点击登录。接下来就是填写详细的姓名、邮箱、密码等信息。这一步简单，没什么好说的。

2、关于信用卡

在注册AWS的时候，是需要填写信用卡信息，AWS会收取1美元的费用，作为验证信用卡的可用性。这里大家需要正确的填写自己的信用卡信息。

3、手机号验证（自己没有通过）

这一步是我最难搞定的一步。AWS的手机号验证流程是这样的：

  1. 在手机验证页面填写好国家和手机号，然后点击下一步
  2. 页面上将展示一个四位数的PIN码，这时候稍等一会
  3. 你的手机将接到一个来自AWS的自动电话
  4. 接通后会有自动语音先说一段英语，然后提示你输入页面上看到四位PIN码
  5. 这时候在手机上输入即可

我通过这种方式实验了两次都没有成功，总是在输入PIN码时，电话就自动挂断了。当我尝试再次验证的时候已经提示我无法处理我的请求，请联系客服/(ㄒoㄒ)/~~。

4、人工验证（通过）

如果你和我一样通过步骤3无法验证通过的话，不要急。我们可以根据提示联系客服。直接在支持页面：https://console.aws.amazon.com/support/home，创建一个问题，点击Create Case，然后输入问题说明自己无法通过自动电话验证通过，例如我当时是这样写的，并附上了我的截图：

When I input my phone number and want to verify the PIN code，I got the error please see my attachment.can you help me?

AWS相应还是很快的，不过毕竟还有时差的存在，大概1-2天你就会收到一封邮件，邮件里会有一个OTP号码。请记下这个号码。稍后你就会接到来自AWS的人工客服电话，我接到是来自西雅图的电话，主要意思就是让你把邮件里的OTP号码，读给他听的。他听到你正确读出OTP号码后，就OK了。这一步就可以通过了。

下图就是我当时收到的邮件：

通过了手机验证这一步，之后就简单了，按照提示进行就行。一般注册全部完成后，24小时之内就可以激活账号，然后就可以创建服务器了。

5、创建EC2实例

首先是区域的选择，AWS提供的区域包括这些：

  • 美国东部 (弗吉尼亚北部)
  • 美国东部 (俄亥俄)
  • 美国西部 (加利福尼亚北部)
  • 美国西部 (俄勒冈)
  • 欧洲 (爱尔兰)
  • 欧洲 (法兰克福)
  • 亚太区域 (东京)
  • 亚太区域 (首尔)
  • 亚太区域 (新加坡)
  • 亚太区域 (悉尼)
  • 亚太区域 (孟买)
  • 南美洲 (圣保罗)

大家可以通过这个网站来从自己所在地ping这些不同的区域，看看那个速度最好。

亚马逊云Ping网站：http://www.cloudping.info/ 我这里测试是首尔和东京的最好，所以选择了东京区域。

然后就是选择系统镜像，这里我选择的是默认的64位的AMI系统

在之后的步骤里按照提示安装完成即可。

6、远程到AWS云主机

安装好系统后，我们都迫不及待的想通过SSH远程连接到我们的云主机。但是在刚安装好的AWS是无法连接到，甚至无法ping通公开IP。原因是AWS有个安全组的配置，这个安全组的配置定义了一些对云主机的入站出站规则，确定哪些IP可以通过哪些端口访问到云主机。默认都是关闭的。所以我们要修改安全组配置。可以查看这里进行设定。

AWS的远程连接，我使用的是xshell工具，使用其他客户端可能涉及到密钥的转化，比较麻烦。由于AWS使用的是pem签名文件，所以在xshell中创建连接时，用户密钥那里要点击浏览，然后导入你再AWS创建实例过程中保存下来的pem文件。用户名固定是ec2-user。连接成功后可以自己修改root用户密码，使用root登录。
