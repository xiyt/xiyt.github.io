利用GitHub备份Sublime Text 3的配置和插件

为了能够保持和备份自己辛苦打造好的Sublime Text  3的相关插件和配置，我的想法了是利用GitHub打造一个自己专属的，可以在其他电脑上随时使用的个性化版本。

1、下载免安装版本的ST3，然后解压缩文件夹，官网下载

2、参考我的这篇文件进行破解激活：Sublime Text 3 最新版3176激活方法

3、安装好自己心爱的各种插件，并且配置好主题、快捷键等其他的配置

4、GitHub中创建好仓库，这一步不用多说

5、关键点来了：需要过滤掉哪些文件或文件夹呢？

这里就涉及到gitignore的编写了，除了Data目录之外，其他的目录都是本来就有的，所有都要上传。而Data目录是我们运行起来ST之后，自动创建的，这里的里面的东西需要分清楚哪些是需要上传到GitHub的：

  • /Data/Cache/，这个是缓存目录，可以忽略
  • /Data/Index/，这个是索引目录，可以忽略
  • /Data/Backup/，这个是备份目录，可以忽略
  • /Data/Local/，这个是本地session目录，可以忽略
  • /Data/Installed Packages，安装的插件目录，需要上传
  • /Data/Packages，插件目录，需要上传

除了这些以外，其他的都是与已经安装插件，和自定义设置有关的数据，都需要上传到GitHub，所以gitignore文件如下：

/Data/Cache/
/Data/Index/
/Data/Backup/
/Data/Local/

6、最后，直接push到GitHub即可。之后如果有变动定期上传即可。如果更换了电脑，也不用害怕需要重新配置ST，直接从GitHub上下载即可。
